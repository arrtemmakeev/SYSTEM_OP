# I - Dependency Map

## Роль файла

Фиксирует зависимости между модулями проекта. Нужен, чтобы не двигаться дальше, если базовый узел не закрыт.

## Критические зависимости

| Если не закрыто | Нельзя финализировать | Чаты |
|---|---|---:|
| Рынок и район | сценарий входа | 01→04 |
| Конкуренты | оффер и цены | 02→04/27 |
| MVP спроса | маркетинг и запуск | 03→28/39 |
| Меню | техкарты | 08→09 |
| Food cost | P&L и цены | 09→24/25/27 |
| Помещение | CAPEX | 11→13/26 |
| Мощность кухни | маркетинг | 14→28 |
| SLA доставки | запуск | 15→39 |
| Санитария | Launch Ready | 20→55 |
| CRM и касса | retention | 21→32/34 |
| P&L | финальное решение | 24→55 |
| Unit economy | масштабирование | 25→40/41 |
| Cash-flow | дата запуска | 26→39/55 |
| Stress-test | GO | 38→55 |

## Правило зависимости

Если базовый модуль не закрыт, следующий модуль может дать только предварительный вывод.

## Статусы зависимости

- OPEN — зависимость не проверена.
- PARTIAL — часть данных есть.
- BLOCKED — движение остановлено.
- CLOSED — можно идти дальше.

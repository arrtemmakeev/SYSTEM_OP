# C - Conditional Routing Map

## Роль файла

Это карта условной маршрутизации. Проект не идёт линейно вслепую. Каждый чат может остановить цепочку.

## Базовая цепочка

00 → 01 → 02 → 03 → 04 → 05 → ... → 49

Контрольный слой активен с первого дня:

50 → 51 → 52 → 53 → 54 → 55 → 56

## Правила остановки

```text
Если 01 = HOLD → не идти в 02.
Если 02 = FIX → вернуться в 01 или 04.
Если 03 = HOLD → не утверждать сценарий входа.
Если 04 = FIX → не запускать бренд, меню и финансы.
Если 09 = FIX → не считать 24/25/27 как финальные.
Если 11 = HOLD → не утверждать 13.
Если 14 = FIX → не масштабировать 28.
Если 20 = HOLD → 39 не может дать GO.
Если 24–26 = NO-GO → маркетинг не масштабировать.
Если 32–34 = FIX → 40/41 не активировать.
Если 55 = HOLD/FIX/NO-GO → запуск запрещён.
```

## Обязательный пакет возврата

1. Что заблокировано.
2. Почему заблокировано.
3. Какой чат исправляет.
4. Какие данные нужны.
5. Какой срок проверки.
6. Какой следующий Decision Gate.

## Главный принцип

Маршрутизация должна защищать проект от красивого, но опасного движения вперёд.

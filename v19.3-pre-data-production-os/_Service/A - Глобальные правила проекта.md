# A - Глобальные правила проекта

## Роль файла

Этот файл задаёт общий режим работы проекта v19.3 Pre-Data Production OS.

## Базовый контекст

Проект: dark kitchen с небольшой зоной самовывоза.
Продукт: роллы, пицца, WOK, сеты, комбо, допродажи.
Рынок: РФ.
Food cost: цель 25%, максимум 28%.
CAPEX: 4 000 000 ₽ с резервом.
Repeat: зрелая цель 45–50%.
Мотивация: 70% оклад + KPI + переменная часть.
Корпоративный канал исключён.

## Главный принцип

Каждый вывод должен вести к одному из четырёх результатов:

1. управленческое решение;
2. расчёт;
3. действие;
4. проверка гипотезы.

## Запреты

- Не выдавать гипотезы как факты.
- Не ставить GO без данных.
- Не запускать маркетинг, если кухня не выдерживает SLA.
- Не считать food cost отдельно от упаковки, доставки, CAC и скидок.
- Не делать приложение до доказанного спроса, CRM, repeat и LTV.
- Не масштабировать проект до стабильной первой точки.
- Не возвращать корпоративный канал в архитектуру.

## Decision Gate

- GO — можно двигаться дальше.
- HOLD — данных недостаточно.
- FIX — проблема исправима, но двигаться дальше рано.
- NO-GO — запуск или решение опасны.

## Формат любого ответа

1. Короткий вывод.
2. Факты.
3. Расчёты.
4. Гипотезы.
5. Риски.
6. Красные зоны.
7. 3 сценария.
8. Action Rules.
9. Decision Gate.
10. Пакет передачи дальше.

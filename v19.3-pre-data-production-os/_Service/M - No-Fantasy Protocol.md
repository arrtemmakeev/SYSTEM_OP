# M - No-Fantasy Protocol

## Роль файла

Защищает проект от красивых, но неподтверждённых выводов.

## Основные правила

- Нет данных → HOLD.
- Нет источника → не факт.
- Нет расчёта → гипотеза.
- Нет связи с деньгами → слабый ответ.
- Нет Decision Gate → ответ не принят.
- Нет следующего шага → ответ бесполезен.
- Нет владельца задачи → задачи не существует.
- Нет срока → действие неуправляемо.

## Запрещённые формулировки

- “Скорее всего рынок примет”.
- “Можно запускать”.
- “Наверное, хватит бюджета”.
- “Повторы будут за счёт качества”.
- “Конкуренты слабые”, если нет контрольных заказов.
- “Food cost нормальный”, если нет техкарт.
- “Маркетинг окупится”, если нет CAC/LTV.

## Правильный формат

Вместо слабого вывода:

> Рынок перспективный.

Пиши:

> Факт: есть район с новыми ЖК.  
> Гипотеза: спрос может быть достаточным.  
> Риск: нет теста спроса.  
> Decision Gate: HOLD.  
> Следующий шаг: чат 03 — MVP и тест спроса.

## Главный принцип

Лучше честный HOLD, чем красивый ложный GO.

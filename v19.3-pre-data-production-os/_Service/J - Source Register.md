# J - Source Register

## Роль файла

Единый реестр источников данных проекта.

## Что заносить в реестр

- договоры;
- КП;
- прайсы поставщиков;
- скрины конкурентов;
- контрольные заказы;
- CRM/POS выгрузки;
- фото/видео факты;
- отзывы;
- карты;
- финансовые таблицы;
- legal/sanitary документы.

## Карточка источника

```text
ID:
Название:
Тип:
Дата:
Владелец:
Ссылка/путь:
Свежесть:
Надёжность:
Применимость к району:
Связанные чаты:
Статус:
Комментарий:
```

## Confidence Score

| Критерий | Балл |
|---|---:|
| Источник официальный | 0–3 |
| Данные свежие | 0–3 |
| Можно проверить | 0–2 |
| Применимо к району | 0–2 |

## Интерпретация

- 8–10: можно использовать для решения.
- 5–7: использовать осторожно.
- 0–4: только гипотеза.

## Правило

Любой вывод, который влияет на деньги, запуск, поставщиков, меню, помещение или legal/sanitary, должен иметь источник в этом реестре.

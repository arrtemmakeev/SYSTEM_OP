# N - Real Data Intake Checklist

## Роль файла

Чек-лист реальных данных, без которых проект не может перейти из Architecture Ready в Data Ready.

## Главный принцип

Пока данные не собраны, любые финансовые, маркетинговые и операционные выводы остаются гипотезами.

## Обязательные данные

| Блок | Что нужно | Статус |
|---|---|---|
| Город | точный город запуска | OPEN |
| Район | приоритетная зона | OPEN |
| Радиус | карта доставки | OPEN |
| Конкуренты | минимум 20–30 игроков | OPEN |
| Цены | реальные цены рынка | OPEN |
| Отзывы | боли клиентов | OPEN |
| Контрольные заказы | вкус, упаковка, SLA | OPEN |
| Помещение | аренда, метраж, условия | OPEN |
| Поставщики | КП и прайсы | OPEN |
| Меню | стартовая матрица | OPEN |
| Техкарты | граммовки и себестоимость | OPEN |
| Упаковка | цена и тест доставки | OPEN |
| Оборудование | КП и CAPEX | OPEN |
| ФОТ | штатка и графики | OPEN |
| CRM/Касса | схема учёта | OPEN |
| Legal | юрформа, налоги, ККТ | OPEN |
| Sanitary | ХАССП, ППК, медкнижки | OPEN |
| Финансы | P&L, unit, cash-flow | OPEN |
| Маркетинг | CAC-гипотезы и каналы | OPEN |
| Retention | события CRM и повтор | OPEN |

## Карточка сбора данных

```text
Данные:
Источник:
Дата:
Ответственный:
Статус:
Достаточно для решения: да/нет
Комментарий:
Связанный чат:
```

## Decision Gate

- Если закрыто меньше 50% данных → HOLD.
- Если закрыты рынок, помещение, меню, техкарты, финансы → можно Data Ready Candidate.
- Если нет legal/sanitary → Launch Ready запрещён.

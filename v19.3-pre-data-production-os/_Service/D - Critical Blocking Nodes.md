# D - Critical Blocking Nodes

## Роль файла

Фиксирует узлы, которые могут заблокировать дальнейшее движение проекта.

## Блокирующие узлы

| Узел | Чаты | Что блокирует |
|---|---:|---|
| Спрос | 01–03 | запуск |
| Сценарий входа | 04 | бренд, меню, финансы |
| Продукт и вкус | 08–09 | продажи, repeat |
| Food cost | 09 | P&L, цены, unit |
| Помещение | 11 | CAPEX, оборудование |
| Мощность кухни | 12–14 | маркетинг, SLA |
| Доставка | 15 | repeat, отзывы |
| Санитария | 20 | запуск |
| IT / касса / CRM | 21–22 | заказы, учёт, repeat |
| Финансы | 24–26 | запуск |
| Цена | 27 | маржа, спрос |
| Retention | 32–34 | рост и масштабирование |
| Риски | 38 | GO |
| План запуска | 39 | дата старта |
| Финальное решение | 55 | запуск |
| AI QA | 56 | качество выводов |

## Правило

Если блокирующий узел не закрыт, следующий этап не получает GO. Максимальный статус — HOLD или FIX.

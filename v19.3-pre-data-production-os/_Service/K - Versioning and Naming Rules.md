# K - Versioning and Naming Rules

## Роль файла

Фиксирует правила именования, версий, статусов и хранения файлов проекта.

## Базовая структура

```text
v19.3-pre-data-production-os/
├── 00-56 Чаты/
├── _Service/
├── _Excel/
├── _Word/
└── _Audit/
```

## Версии

| Версия | Статус | Смысл |
|---|---|---|
| v19.3 | Pre-Data | архитектура до реальных данных |
| v19.4 | Data Ready Candidate | данные собраны и проверяются |
| v20.0 | Launch Ready Candidate | можно обсуждать запуск |

## Статусы файлов

| Статус | Значение |
|---|---|
| DRAFT | черновик |
| REVIEW | на проверке |
| APPROVED | утверждено |
| ARCHIVED | устарело |

## Правила именования

```text
[Блок] - [Название] - [Версия] - [Статус]
```

Пример:

```text
Финансы - P&L модель - v19.4 - REVIEW.xlsx
```

## Запреты

- Не использовать названия `финал`, `финал2`, `последний`, `новый финал`.
- Не перезаписывать утверждённый файл без новой версии.
- Не хранить расчёты без источников.
- Не смешивать гипотезы и факты в одном файле без маркировки.

## Правило обновления

Любое обновление должно фиксировать:

- дату;
- владельца;
- что изменено;
- почему изменено;
- какой файл заменён;
- какой Decision Gate затронут.
